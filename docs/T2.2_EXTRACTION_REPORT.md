# T2.2 extraction and fork reconciliation report

Date: 2026-07-18

## Outcome

The 32-voice `digital-beacon` Shaper is now the single packaged implementation
under `src/harmonic_shaper`. It runs with `python -m harmonic_shaper` or the
`harmonic-shaper` entry point. The direct v1 OSC listener remains exactly
wire-compatible with `/digital/*` on UDP 9002. NaturalHarmony broadcast input
under `/beacon/*` on UDP 9001 remains implemented but is disabled unless the
process is started with `--slave`.

`/shaper/*` is named throughout configuration and documentation as the planned
native namespace. No `/shaper/*` OSC aliases were added: that wire change needs
a later contract bump.

## NaturalHarmony reconciliation

One useful behavior had been lost in the digital-beacon fork: the original
`Minilab3Control` and its CC mapping arrays. It has been migrated and is started
alongside the Launchpad controller.

The original also contains a `DatasetLogger`, experiment-session HTTP endpoints,
and an older static dashboard. Those are experiment capture/UI tooling rather
than the Shaper runtime requested by T2.2, so they were reviewed but deliberately
not migrated. The older `/shaper/*` receive table was also not restored because
the binding T1.3 contract says the implemented v1 wire surface is `/digital/*`.
No engine or state behavior in the five-voice original supersedes the evolved
32-voice fork.

## Clipping-suspicious paths for T2.4

These are observations only; the engine was not redesigned in this extraction.

1. `1/sqrt(N)` controls expected/RMS growth, not worst-case peak growth. Voices
   with aligned phases can still sum coherently to approximately `sqrt(N)` after
   normalization before the final limiter.
2. Per-voice gain LFO applies `gain * (1 + lfo_gain * lfo_mod)`, which can double
   a voice. The normalization count does not include this gain increase.
3. The negative sidechain branch computes
   `1 + abs(sidechain_amount) * (1 - beacon_level)`. At `-1` and a zero beacon
   level this is a 2x boost, despite configuration describing negative values as
   ducking. This is the clearest unexpected pre-limiter gain source.
4. The recording tap is intentionally before `tanh`; captured blocks can exceed
   full scale even though real-time output is bounded by
   `tanh(mix * 1.05) * 0.95`.
5. Waveshaping normalizes each shaped voice back near unit peak before summing.
   It changes crest factor and harmonic correlation, while the mix normalization
   still depends only on voice count.
6. `synthesize_prepared()` likewise returns an unbounded raw sum by design. Its
   CLI applies soft clipping/normalization later, so callers using the function
   directly must perform that final stage themselves.

The headless render regression intentionally uses one reference voice and asserts
finite samples with peak at or below 1.0. Multi-voice peak behavior remains the
subject of T2.4 rather than being silently changed here.
